운영체제(OS)

    각 하드웨어에 적절한 전기신호를 흘려주는 소프트웨어이다.
    실행 가능한 프로그햄과 하드웨어를 연결해주는 역할을 한다.

Linux®(리눅스)

    Linux®(이하 Linux)에서의 ®은 registered trademark symbol로서, 등록된 상표임을 알리기 위해 사용한다.
    Linux는 오픈소스(Open Soure Software, OSS) 운영체제(OS)이다.
    이 때, 오픈소스는 공개적으로 접근할 수 있게 설계되어 누구나 자유롭게 확인, 수정, 배포할 수 있는 코드를 의미한다.

오픈소스(Open Source)

    Linux GNU GPL(GNU General Public License)를 사용한다.
    GNU GPL이란, 자유 소프트웨어 재단에서 만든 자유 소프트웨어 라이선스를 의미한다.
    즉, 누구든지 소스코드를 실행, 연구, 수정, 재배포할 수 있다.
    동일한 라이선스가 유지되는 한 수정힌 코드의 복사본을 판매할 수도 있다.

Linux OS 종류

    https://upload.wikimedia.org/wikipedia/commons/1/1b/Linux_Distribution_Timeline.svg

1. 슬랙웨어(Slackware) 계열

        가장 오래된 배포판으로서 현재까지도 운영되고 있다.
        1개의 패키지를 설치하고자 할 때, 필요한 다른 패키지까지 설치해주지 않기 때문에 초보자가 사용하기 어렵고, 현재는 다른 배포판에 비해 많이 사용하지 않는 추세이다.

2. 데비안(Debian) 계열

        오픈소스 성격이 강하기 때문에, 파생된 배포판이 다양하다.
        다른 배포판에 비해 안정성이 매우 높으며 다양한 패키지를 활용할 수 있다.
        데스크탑으로 사용하기 보다는 서버로 운영하는 것이 더 적합하다.
    
        * Ubuntu
        - 리눅스 배포판 중에 가장 많이 사용되고 있는 배포판이다.
    
        * 칼리 리눅스(Kali Linux)
        - 해킹에 관련된 패키지가 대다수이며, 고급 침투 테스트 및 보안 검사에 특화된 배포판이다.
        - 우분투 기반인 백트랙(BackTrack) 배포판도 있다. 백트랙은 일명 해킹머신이라고 부른다.

3. 레드헷(Red Hat) 계열

        기업용 서버 운영체제로 가장 많이 사용되고 있다.
        레드헷 리눅스(RHEL, RedHatEnterpriseLinux)는 보통 무료가 아닌 유료버전을 의미한다.
        
        * 페도라(Fedora)
        RHEL에 추가될 신규 기술을 미리 테스트해볼 수 있는 베타 버전 배포판이다.
        신기술을 체험해볼 수는 있으나 해당 부분에 버그나 에러가 많은 편이다.
        전문가가 사용하기 좋은 배포판이다.
        페도라에서 안정화된 기술들은 RHEL에 포함된다.

CLI(Command Line Interface)

    키보드를 이용해 운영체제와 상호작용하는 방식을 말한다.
    접근 권한 등을 통해 해당 컴퓨터에 무제한으로 접근할 수 있는 강력한 방법이다.

GUI(Graphical User Interfaces)

    마우스로 가리키고 클릭하는 방식의 그래픽 사용 방식을 맗한다.
    하드웨어에 직접적으로 요청하기 어렵지만, CLI에서 기본적으로 사용하는 부분들을 시각적으로 편하게 사용할 수 있다.

리눅스 커널 

    커널(Kernel)
    - 복숭아 같은 과일의 씨앗으로써, 핵심이라는 뜻이다.
    - 리눅스 커널(씨앗)은 Linux 운영체제(복숭아) 안에 위치해 있고 Linux 운영체제의 주요 구성 요소이자 하드웨어와 프로세스(실행중인 프로그램)을 이어주는 핵심 인터페이스(틀)이다.
    - 컴퓨터 유형에 관계없이(전화기, 노트북, 서버 등) 하드웨어의 모든 주요 기능을 제어한다.
    - 올바르고 구현된 커널은 사용자가 볼 수 없으며 커널공간이라는 자신만의 작은 작업공간에서 메모리를 할당하고, 저장된 모든 항목을 추적한다.
    - 반대로 웹 브라우저 및 파일처럼 사용자가 볼 수 있는 것을 사용자 공간이라 한다.
    - 사용자 공간(사용자 프로세스)과 하드웨어를 연결해주기 위해 여러 일을 하는 관리자가 바로 커널이다.

    1. 메모리 관리
    2. 프로세스 관리
    3. 장치 드라이버
    4. 시스템 호출 및 보안

1. 전원인가

2. BIOS

        컴퓨터에 전원이 인가되면 실행되는 최초의 프로그램이다.
		디바이스를 초기화하고 검사 (POST, Power On Self Test)하는 역할을 수행하고
		Boot Loader와 운영 체제를 메모리(RAM, Random Access Memory)로 올리는 기능을 수행한다.

        - BIOS(Basic Input/Output System) 또는 UEFI(Unified Extensible Firmware Interface) 로드
   
            ※ BIOS: 전통적인 PC 펌웨어
			※ UEFI: BIOS를 조금 더 정형화하고 표준화한 PC 펌웨어, 2TB 이상 Storage 지원, 더 빠른 부팅 시간, UI 및 기능 개선 등
			※ 펌웨어: 하드웨어를 제어하는 가장 기본적인 프로그램
			※ PC 환경에서는 UEFI가 표준화되어 가고 있지만, 가상화 환경에서는 BIOS가 여전히 많이 사용된다.

        - 하드웨어 검사
        - 부팅 기기 선택
        - EFI 시스템 파티션 식별

3. Boot Loader

        - 부트 로더(Boot Loader)는 컴퓨터 시스템이 부팅될 때 최초로 실행되는 프로그램이다.
        - 대부분 리눅스 배포판에서 기본 부트로더로 GRUB(Grand Unified Boot Loader)을 사용한다.
        - 운영 체제를 실행하기 전에 하드웨어 초기화, 시스템 구성, 운영 체제 로딩을 담당한다.
		- 여러 운영체제를 불러온 경우 어떤 것을 부팅할지 사용자가 선택할 수 있도록 UI를 제공한다.

        즉, 컴퓨터의 전원이 켜질 때부터 운영 체제가 동작할 때까지의 초기화 및 준비과정을 담당하는 중요한 소프트웨어이다.

        - Boot Loader 로드
        - 부팅할 커널 선택
        - 커널 로드

4. Linux

       운영체제는 각 하드웨어에 알맞는 전기신호를 보내주는 역할을 한다.

       - 커널 자료구조 초기화
       - init/systemd 시작
       - 시작 스크립트 실행
       - 시스템 실행
